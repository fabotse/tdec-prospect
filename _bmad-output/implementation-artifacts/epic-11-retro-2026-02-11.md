## Challenges & Patterns Identificados

### 1. Sem Build Gate — 6 Erros de TypeScript Escaparam
- `useRef` sem argumento inicial, tipos divergentes, deep type instantiation, `unknown` em JSX
- Apenas detectados na story 11.8 quando `npm run build` foi executado
- Se 11.8 nao existisse, esses erros so seriam pegos no deploy
- **Licao:** `npm run build` precisa ser step obrigatorio no workflow dev-story

### 2. .gitignore CRITICAL — .env.example Bloqueado
- `.gitignore` linha 31 tem glob `.env*` que faz match com `.env.example`
- Arquivo template nunca seria commitado — perda de documentacao
- Detectado apenas durante code review da 11.8
- **Fix:** Adicionado `!.env.example` ao `.gitignore`
- **Licao:** Stories de deploy devem incluir auditoria de `.gitignore`

### 3. Cache Key Inconsistente (11.7)
- Hook usava `leadTracking` (camelCase) mas query key esperava `lead-tracking` (kebab-case)
- Causava invalidacao de cache falhar silenciosamente
- Detectado no code review
- **Licao:** Estabelecer convencao de naming para cache keys (kebab-case)

### 4. formatPhone Duplicado
- Funcao `formatPhone` implementada em `use-whatsapp-bulk-send.ts` E em `whatsapp-actions.ts`
- Mesma logica, dois locais de manutencao
- **Licao:** Consolidar em utilidade compartilhada

### 5. PT-BR Acentos Recorrentes (11.6)
- Strings UI sem acentos corretos detectadas no code review
- Problema recorrente desde epics anteriores
- **Licao:** Incluir verificacao de acentos no preflight checklist

### 6. Rastreabilidade de Action Items
- Retro da Epic 10 definiu 8 action items
- Items 3 (teste flaky) e 4 (auth unification) foram resolvidos via commit `e2b0873`
- Porem nao foram documentados no artefato da retro — so descobertos por investigacao do git log
- Item 1 (dev preflight checklist) e items 5-8 nao foram feitos
- **Licao:** Action items precisam de evidencia concreta (link para commit/PR)
