## Desafios e Aprendizados

### 1. API Instantly com tipos divergentes da documentacao
**Problema:** A documentacao diz que `esp_code` e string ("Google", "Microsoft"), mas em producao veio como numero (1, 3, 999). `esg_code` idem. `status_summary` veio como objeto em vez de string.

**Impacto:** Story 14.5 precisou alterar tipos (`InstantlyLeadEntry`) e service (`mapToLeadTracking`) — algo que a 14.1 explicitamente dizia para NAO fazer. Criacao nao-planejada de `ESP_CODE_MAP`, `ESG_CODE_MAP`, `resolveEspCode()`, `resolveEsgCode()`, `resolveStatusSummary()`.

**Causa raiz:** Tipos definidos com base na documentacao da API externa sem validacao com dados reais.

**Aprendizado:** Documentacao de API externa != realidade. Sempre validar com chamada real antes de definir tipos TypeScript.

### 2. Bug off-by-one no step numbering (14.7)
**Problema:** API route `/steps` convertia `position + 1` assumindo que Instantly usava indexacao 1-based. Na verdade e 0-based. Step 0 nunca encontrava match no stepsMap.

**Causa raiz:** Suposicao sobre indexacao sem validacao com dados reais — mesmo padrao do problema anterior.

**Fix:** Removido `+ 1`, stepsMap agora 0-based, `formatStep()` exibe 1-based para o usuario.

### 3. Mudancas nao-relacionadas vazando no escopo
`Sidebar.tsx` (refator hydration) apareceu nos commits das stories 14.2 e 14.4. Nada grave, mas polui escopo e dificulta code review.

### 4. Retrospectivas puladas (Epics 12 e 13)
Ultima retro foi Epic 11 (2026-02-11). Aprendizados de 3 epics inteiros nao foram documentados.

### 5. Pre-existing TS build errors (14.3)
3 erros TypeScript em arquivos nao-relacionados foram encontrados e corrigidos durante a 14.3 (insights route, UsageServiceName, apollo.ts). Scope creep necessario para build passar.
