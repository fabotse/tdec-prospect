## Desafios e Aprendizados

### 1. Error handling silencioso — padrao recorrente (3/6 stories)

**Problema:** Fetches sem check de `response.ok`, promises sem try/catch, erros engolidos sem feedback ao usuario.
- 16.3: sendAgentMessage e saveBriefing engoliam erros silenciosamente
- 16.5: handleConfirmPlan e handleCancelPlan sem try/catch adequado
- 16.6: Mesmos padroes herdados

**Impacto:** Na Epic 16, erro silencioso = toast faltando. Na Epic 17, erro silencioso no pipeline = dados inconsistentes no banco.

**Aprendizado:** Convenção de error handling deve ser obrigatoria e documentada antes da Epic 17.

### 2. Non-null assertions repetidas (2/6 stories)

**Problema:** Uso de `!` (non-null assertion) ao inves de null guards explicitos.
- 16.3: `execId!` no AgentChat
- 16.6: `state.briefing!` e `state.productMentioned!` no useBriefingFlow

**Aprendizado:** Banir `!` assertions. Usar `if (!x) return;` guards. Considerar ESLint rule `@typescript-eslint/no-non-null-assertion`.

### 3. Testes de fluxos alternativos insuficientes

**Problema:** Stories entregues com testes focados no happy path, fluxos de erro descobertos apenas no code review.
- 16.3: Task 8.5 (testes briefing flow no AgentChat) nao implementada — 7 testes adicionados no review
- 16.6: Teste de briefing incompleto + produto mencionado ausente

**Aprendizado:** Testes de fluxos de erro devem ser requisito explicito, nao opcional.

### 4. Bugs de integracao entre stories

**Problema:** Rota de mensagens (16.2) rejeitava role 'agent', bloqueando a 16.3. Mensagem duplicada quando parse falhava.

**Aprendizado:** Ao criar APIs, antecipar roles/usos futuros com whitelist. Sempre verificar fluxos de erro entre stories.
